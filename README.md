# reMarkable 管理工具（GUI 版）

</div>

一个面向 reMarkable 电子墨水平板的图形化管理工具，支持字体/壁纸上传、时间同步、设备控制以及文档预览与导出等功能。新版本提供现代化 GUI 体验，并能轻松打包成单文件可执行程序。

## 功能亮点

- **一键连接**：支持 USB 与 Wi-Fi 两种连接方式，自动记忆 root 账号信息。
- **字体管理**：直接从文件对话框选择字体（TTF/OTF），可选自动重命名为 `zwzt.ttf` 并上传。
- **壁纸处理**：上传任意常见格式图片，自动转换为 1620×2160 分辨率，可预览并选择是否保持原始纵横比。
- **时间设置**：同步本地时间、查看设备时间信息或一键将时区设为东八区。
- **设备控制**：支持重启设备、启用 SSH 服务、提升前光亮度并创建持久化服务。
- **文档预览与导出**：自动识别设备中的笔记、PDF、EPUB 等文件，显示基础元数据并支持导出到本地。

## 运行环境

- Python 3.9 及以上版本（建议使用 64 位环境）。
- 依赖库：`paramiko`、`PyQt5`、`Pillow`。

```bash
pip install -r requirements.txt  # 或手动安装上述依赖
```

> **提示**：如果你计划打包成单文件可执行程序，推荐在干净的虚拟环境中安装依赖。

## 快速开始

1. 克隆或下载本仓库。
2. 在终端中运行：

   ```bash
   python rmtool.py
   ```

   Windows 用户可以继续使用仓库中的 `rmtool.bat` 启动脚本。

3. 首次连接时在界面左上角填写设备地址与 root 密码即可。连接成功后即可访问全部功能，无需手动创建 `fonts` 或 `wallpaper` 目录。

## 打包为单文件可执行程序

项目已针对 PyInstaller 进行了优化。示例命令如下：

```bash
pyinstaller --noconsole --windowed --onefile rmtool.py
```

打包完成后，可执行文件会出现在 `dist/` 目录中，直接拷贝到目标机器即可运行。

## 常见问题

- **连接失败？** 请确认 reMarkable 已开启开发者模式并允许 SSH 访问，且 USB/Wi-Fi 连接稳定。
- **上传壁纸后显示异常？** 可尝试关闭“保持纵横比”选项重新上传，或在设置中恢复备份的 `suspended.png.backup`。
- **无法导出文档？** 目前仅支持导出已在设备上生成的 PDF/EPUB/ZIP/NOTE/RM 文件。原生笔记会导出对应目录，需借助第三方工具进一步转换。
