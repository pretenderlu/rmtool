# rmtool v1.4.1

## English

> macOS build fixes: the app now opens normally when double-clicked, and the window no longer fills the whole screen on launch.

### macOS

- Fixed the app failing to open when double-clicked: an unsigned app
  launched from Finder runs under Gatekeeper app translocation on a
  read-only mount, and creating the state directory beside the bundle
  crashed the app before any window appeared. App state (config, logs,
  known hosts) now lives in `~/Library/Application Support/rmtool`.
- Logging setup can no longer abort startup: if the state directory is
  not writable, logging falls back to stderr instead of crashing.
- The main window now defaults to about 80% of the usable screen instead
  of a fixed 1760×1100 that exceeded MacBook screens and was clamped to
  look fullscreen. Bounds stay at 1280×900 (min) to 1760×1100 (max).

## 中文

> macOS 构建修复：双击可正常打开，启动时窗口不再占满整个屏幕。

### macOS

- 修复双击无法打开的问题：未公证的 app 从 Finder 启动时会在 Gatekeeper
  转移机制的只读挂载点上运行，之前在 bundle 旁边创建状态目录直接导致
  启动崩溃。应用状态（配置、日志、known hosts）现在存放在
  `~/Library/Application Support/rmtool`。
- 日志初始化不再会导致启动失败：状态目录不可写时降级为终端输出。
- 主窗口默认尺寸改为屏幕可用区域的约 80%，不再是超出 MacBook 屏幕、
  被钳制成"全屏"的固定 1760×1100；尺寸范围保持 1280×900（最小）至
  1760×1100（最大）。
