# rmtool v1.3.2

## English

> Hotfix release: worker callbacks no longer touch widgets after their tab was destroyed, fixing intermittent aborts on macOS (also hardens against deferred signal delivery everywhere).

This release is a full interface modernization: a cleaner minimal design, a new
sidebar navigation, and a native dashboard that no longer depends on
QtWebEngine — plus a frequently requested option to blank the sleep-screen
carousel, safely.

### Modernized interface

- Single design-token source drives every color, radius, spacing, and font
  size in both dark and light themes; one accent color replaces the old
  gradients, and buttons now follow a clear primary / secondary / danger
  hierarchy.
- New sidebar navigation with five flattened top-level pages: Dashboard,
  Wallpapers, Documents, Fonts, and Device Tools. Font management is now a
  first-class page instead of a nested section.
- The dashboard is reimplemented with native Qt widgets. The QtWebEngine
  dependency — and its large package footprint — is gone.
- Unified six-step type scale with proper high-DPI scaling, so text and
  controls stay crisp and correctly sized on 4K displays; sidebar icons are
  now resolution-independent SVG renderings.
- Wallpaper and document previews render at physical pixel resolution on
  high-DPI screens.

### Blank sleep carousel

- New wallpaper-tab option: replace the sleep-screen carousel illustrations
  with fully transparent PNGs for a clean sleep screen.
- Original illustrations are backed up on the device first (never
  overwritten), and unchecking the option restores them.
- Uploading a custom suspended wallpaper now uses the same backup-first
  flow, closing the old irreversible-overwrite path.

### Fixes

- Document preview toggle highlight now follows the visible page (cover vs
  metadata).
- Document page defaults to a wider list pane; the search box can no longer
  be squeezed below its placeholder.
- Sidebar connect buttons stack vertically and can no longer clip their
  labels.
- Wallpaper control column reorganized into clear, aligned sections.

## 中文

> 热修复版本：后台任务回调在所属页面已销毁后不再访问界面组件，修复 macOS 上的偶发崩溃（同时增强了各平台对延迟信号派发的健壮性）。

本版本是一次完整的界面现代化改造：更干净的极简设计、全新的侧边栏导航，以及不再依赖
QtWebEngine 的原生仪表盘——另外还加入了呼声很高的"空白替换休眠轮播"功能，且全程可恢复。

### 界面现代化

- 全部颜色、圆角、间距、字号统一由单一设计 token 来源驱动（深浅两套主题）；单一强调色取代
  旧渐变，按钮按 主要 / 次要 / 危险 三级区分。
- 全新侧边栏导航，五个打平的顶层页面：仪表盘、壁纸管理、文档中心、字体管理、设备工具。
  字体管理从嵌套区块提升为独立页面。
- 仪表盘改用 Qt 原生组件重写，彻底移除 QtWebEngine 依赖，安装包体积明显减小。
- 统一的六级字体阶梯 + 完整的高 DPI 缩放支持，4K 高分屏下文字和控件清晰且大小正常；
  侧边栏图标改为与分辨率无关的 SVG 渲染。
- 壁纸与文档预览在高分屏下按物理像素渲染，不再模糊。

### 空白休眠轮播

- 壁纸页新增选项：把休眠轮播插图替换为带透明通道的全透明 PNG，获得干净的休眠屏。
- 替换前自动在设备上备份原始插图（备份不会被覆盖），取消勾选即可恢复。
- 上传自定义休眠壁纸也改为同样的"先备份"流程，堵住了旧版本不可逆覆盖官方插图的问题。

### 问题修复

- 文档预览的"元数据 / 图像预览"高亮现在与实际显示内容一致。
- 文档页默认布局加宽列表侧；搜索框不会再被挤压到占位文字显示不全。
- 侧边栏连接/断开按钮改为上下排列，文字不再被裁切。
- 壁纸页控制区重新整理为分区清晰、控件对齐的布局。
