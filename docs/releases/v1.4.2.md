# rmtool v1.4.2

## English

> Windows window/taskbar icon fix: the app now shows the rmtool icon in the
> title bar and on the taskbar instead of the generic Python icon.

### Windows

- The window icon is now set at runtime from the bundled
  `assets/rmtool-icon.ico`, so the title bar and taskbar button show the
  rmtool icon. Previously only the exe file icon was embedded, and Windows
  fell back to the generic pythonw icon for the running window.
- The process now registers a stable AppUserModelID (`rmtool.app`), which
  Windows requires to attribute the taskbar icon to the app instead of the
  Python runtime.
- The icon file is now bundled into the PyInstaller builds (both the local
  spec and `build-portable.ps1` used by CI).

## 中文

> Windows 窗口/任务栏图标修复：标题栏和任务栏现在显示 rmtool 图标，不再是
> 默认的 Python 图标。

### Windows

- 运行时从打包的 `assets/rmtool-icon.ico` 设置窗口图标，标题栏与任务栏
  按钮现在显示 rmtool 图标。此前只嵌入了 exe 文件图标，运行中的窗口被
  Windows 回退为 pythonw 的默认图标。
- 进程现在注册固定的 AppUserModelID（`rmtool.app`），这是 Windows 将
  任务栏图标归属到应用本身（而非 Python 运行时）的必要条件。
- 图标文件已加入 PyInstaller 打包资源（本地 spec 与 CI 使用的
  `build-portable.ps1` 均已更新）。
