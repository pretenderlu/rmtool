# rmtool v1.4.0

## English

> New KOReader book management page, plus a connection-stability overhaul: post-connect background work is now serialized so the device's SSH server is no longer overwhelmed.

### KOReader book management

- New sidebar page "KOReader" for managing the book library of a KOReader
  installation, which lives outside the native xochitl document system.
- Detects all three installation layouts: official (`/home/root/koreader`),
  Toltec (`/opt/koreader`), and xovi/appload on Paper Pro devices; devices
  without KOReader see a clear notice and nothing is written.
- Start directory follows KOReader's own `home_dir` setting (read-only),
  falling back to `/home/root/books` and then the home directory; the current
  path is shown and can be changed manually.
- Browse subfolders, upload and download books, delete books together with
  their `.sdr` metadata, and create folders — no service restarts needed,
  KOReader picks up changes on its own.
- The xochitl document tree stays a hard forbidden zone; all remote paths are
  shell-quoted.

### Connection stability

- After connecting, background refreshes (documents, wallpaper previews, font
  list) now run strictly one at a time instead of all at once. The device's
  dropbear SSH server no longer drops the connection under the channel burst
  that could hit right after connecting.
- The font list now fills in silently — connecting no longer pops a modal
  progress dialog.
- The KOReader page loads lazily on first visit instead of on connect.
- Disconnecting and reconnecting quickly no longer writes stale data from the
  previous connection into the UI.

## 中文

> 新增 KOReader 书籍管理页面，并重构连接稳定性：连接后的后台任务改为串行执行，设备 SSH 服务不再被并发洪峰压垮。

### KOReader 书籍管理

- 新增侧边栏"KOReader"页面，管理独立于原生 xochitl 文档体系的 KOReader 书库。
- 支持检测全部三种安装形态：官方（`/home/root/koreader`）、Toltec
  （`/opt/koreader`）、Paper Pro 系列的 xovi/appload；未安装 KOReader 的设备
  显示明确提示且不写入任何内容。
- 起始目录跟随 KOReader 自己的 `home_dir` 设置（只读），依次回退
  `/home/root/books` 和用户主目录；界面显示当前路径并可手动跳转。
- 可浏览子目录、上传和下载书籍、连同 `.sdr` 元数据一起删除书籍、新建
  文件夹——无需重启任何服务，KOReader 会自行发现变更。
- xochitl 文档目录为硬性禁写区，所有远端路径均经 shell 转义。

### 连接稳定性

- 连接后的后台刷新（文档、壁纸预览、字体列表）改为严格串行执行，不再
  同时启动。设备 dropbear SSH 服务不会再被连接瞬间的通道洪峰压垮而断开。
- 字体列表改为静默填充——连接后不再弹出模态进度对话框。
- KOReader 页面改为首次切换时懒加载，不再在连接时刷新。
- 断开后快速重连，不会再把上一次连接的旧数据写进界面。
